# Happy Number

## Problem Statement

A **happy number** is defined by the following process:

- Replace the number with the sum of the squares of its digits.
- Repeat the process until:
  - the number becomes `1`, or
  - the process enters a cycle that never reaches `1`.

Return `true` if the given number is happy; otherwise, return `false`.

---

## Intuition

At first glance, this doesn't look like a linked list problem.

But think of every number as pointing to the **next number** generated by the digit-square operation.

For example:

```text
19 ŌåÆ 82 ŌåÆ 68 ŌåÆ 100 ŌåÆ 1
```

or

```text
2 ŌåÆ 4 ŌåÆ 16 ŌåÆ 37 ŌåÆ 58 ŌåÆ 89 ŌåÆ ...
```

This forms a sequence where each value leads to exactly one next value.

If the number isn't happy, this sequence eventually enters a **cycle**.

That means this problem becomes identical to **Cycle Detection in a Linked List**.

So we can apply **Floyd's Tortoise and Hare Algorithm**.

---

## Approach

- Create a helper function that returns the sum of the squares of a number's digits.
- Initialize two pointers:
  - `slow` moves one operation at a time.
  - `fast` moves two operations at a time.
- Continue until both pointers meet.
- If they meet at `1`, the number is happy.
- Otherwise, they have met inside a cycle, so the number is not happy.

## Time Complexity

- **Time:** `O(log n)` per operation, with a bounded number of iterations before reaching `1` or a cycle.
- **Space:** `O(1)`

---

## Key Takeaway

The trick in this problem is recognizing that it secretly asks you to **detect a cycle**.

Every number generates exactly one next number, creating a chain just like a linked list:

```text
Current Number
      Ōåō
Next Number
      Ōåō
Next Number
      Ōåō
...
```

If the sequence ever repeats without reaching `1`, you've entered a cycle.

That's why **Floyd's Cycle Detection Algorithm** works perfectly here, even though there isn't an actual linked list.

A good interview habit is to ask yourself:

> *"Can I think of each state as pointing to exactly one next state?"*
